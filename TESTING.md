# 全体テスト手順

## 1. ビルド確認

```powershell
cd c:\dev\gear-nexus
npm run build
```

- エラーなく完了すれば本番ビルドは問題なしです。

## 2. 開発サーバーで動作確認

```powershell
cd c:\dev\gear-nexus
npm run dev
```

ブラウザで **http://localhost:3000** を開いてください。

---

## 機能チェックリスト

### トップページ (/)
- [ ] ページが表示される
- [ ] 検索フォームでレビュー検索できる
- [ ] **新着レビュー**が横にゆっくりスライドする（3件以上でループ）
- [ ] **人気機材**が横にゆっくりスライドする
- [ ] カルーセルのカードをホバーすると「Xでポスト」ボタンが出る
- [ ] カードクリックでレビュー詳細へ遷移する
- [ ] 楽器・機材ニュース、ライブ予定、地図など他セクションが表示される

### レビュー
- [ ] **レビュー一覧** (/reviews)：一覧表示・検索・カテゴリ
- [ ] **レビュー詳細** (/reviews/[id])：タイトル・本文・画像・いいね・役に立った・比較リスト
- [ ] **Xでポスト**ボタンで X の投稿画面が開く（タイトル＋URLが入る）
- [ ] **新規投稿** (/reviews/new)：レビュー作成
- [ ] **編集**：自分のレビューを編集できる

### 認証・ユーザー
- [ ] **ログイン** (/login)
- [ ] **プロフィール**：初回はオンボーディング、その後プロフィール編集
- [ ] **マイページ**：自分のレビュー・いいね一覧など

### その他
- [ ] **比較リスト** (/reviews/compare)
- [ ] **カスタム手帳** (/notebook)
- [ ] **ブログ**・**フォト**・**イベント**・**ヘルプ**（ナビのリンク先）

### 管理者（管理者ユーザーのみ）
- [ ] **管理** (/admin)：登録ユーザー一覧・レビュー一覧など
- [ ] お知らせ管理は廃止済みのためリンクなし

---

## 注意

- Firebase（Auth / Firestore / Storage）の環境変数が `.env.local` に設定されている必要があります。
- 管理者機能はプロフィールの `user_id` が管理者リストに含まれる場合のみ利用できます。
